# Architecture

## Pattern

Reproducible research data pipeline for educational performance analysis. The project follows a **multi-stage ETL pipeline** pattern common in academic data science: raw source data is ingested and cleaned per source, then merged into a single trusted analytical dataset, which is loaded by an analysis layer that applies statistical and machine learning methods to produce visualizations and reports.

The overall flow is:
```
data/base/ (source files)
  → [ingest.py] →
data/raw/ (per-source CSVs)
  → [transform.py] →
data/trusted/ (merged analytical CSV)
  → [analytics.py / analytics.ipynb] →
reports/ (figures, HTML report, slides)
```

## Layers / Components

### 1. Ingestion Layer — `src/ingest.py`
Reads from raw source files in `data/base/` and produces per-domain CSV files in `data/raw/`. Each data domain (IDEB, Censo Escolar, IBGE, IDHM) has its own builder function:

- `build_indicadores_municipais_ma_ideb_2023()` — reads INEP IDEB Excel, filters Maranhão municipal network
- `build_indicadores_municipais_ma_censo_escolar_2023()` — reads microdados CSV (large file), aggregates school-level features to municipality level
- `build_indicadores_municipais_ma_censo_docentes_2023()` — supplements teacher qualification data from the Sinopse Estatística ODS file
- `build_indicadores_municipais_ma_ibge_2010()` — reads IBGE socioeconomic Excel
- `build_indicadores_municipais_ma_idhm_2010()` — reads HDI Excel

Helper utilities within `ingest.py`: `safe_divide()`, `normalize_municipio_code()`, `resolve_censo_path()`, `ensure_columns()`.

### 2. Transformation Layer — `src/transform.py`
Reads the per-source CSVs from `data/raw/`, cleans and standardizes each dataset, then merges all four into a single analytical dataset at `data/trusted/indicadores_municipais_ma.csv`. Key operations:
- Replaces INEP non-disclosure codes (`ND`, `*`, `--`) with `NaN`
- Drops redundant columns (e.g., state code, individual subject notes)
- Normalizes municipality names for join compatibility (`.str.upper().str.strip()`)
- Performs sequential left-joins: IDEB + Censo (index join) → IBGE (on `NO_MUNICIPIO`) → IDHM (on `NO_MUNICIPIO`)

### 3. Analysis Layer — `src/analytics.py` and `src/analytics.ipynb`
Loads the trusted dataset and executes the full analytical pipeline:

- **IDEB distribution** — histogram with mean/median/std
- **Clustering** — K-Means on `RENDA_PER_CAPITA_2010` and `IDHM_2010` after `RobustScaler` normalization; cluster count selected by silhouette score (default `CLUSTER_K = 2`)
- **PCA visualization** — 2-component PCA for cluster scatter plots comparing k=2,3,4
- **Cluster stability** — bootstrap ARI validation (`_estabilidade_clusters`)
- **Variable relevance** — ablation analysis of clustering variables
- **Intra-cluster profiling** — within each cluster, top/bottom IDEB municipalities by quantile, Mann-Whitney tests across explanatory variables
- **Random Forest** — `RandomForestRegressor` for overall IDEB importance ranking; `RandomForestClassifier` per cluster for identifying high vs low IDEB differentiators
- **Figure export** — all plots saved to `reports/figures/` as PNG at 150 dpi

The notebook `src/analytics.ipynb` mirrors `analytics.py` for interactive execution, with HTML export as the primary report deliverable.

### 4. Utility Layer — `src/util.py`
Provides a simple CLI progress timer using a background thread (`start_timer()` / `stop_timer()`), used during long-running ingestion steps.

### 5. Reporting Layer — `reports/`
- `reports/analytics-executado.html` — executed notebook output (primary deliverable)
- `reports/report.qmd` — Quarto template for a structured analytical report (currently a scaffold)
- `reports/seminario_slides.qmd` — Quarto RevealJS slides presenting methodology and Python concepts
- `reports/figures/` — PNG charts generated by the analytics pipeline
- `reports/theme.scss` — custom RevealJS theme

### 6. Academic Paper — `artigo/`
LaTeX article manuscript with standard IMRaD structure (abstract, introduction, related works, methods, experiments, conclusions, acknowledgments). Compiled to `artigo/main.pdf`.

## Data Flow

```
Sources (Excel/CSV/ODS in data/base/)
  │
  ├─ data/base/divulgacao_anos_iniciais_municipios_2023.xlsx  ─→ IDEB data
  ├─ data/base/microdados_censo_escolar_2023/dados/
  │    microdados_ed_basica_2023.csv                          ─→ school census microdata
  │    docentes.ods                                           ─→ teacher qualification sinopse
  ├─ data/base/ibge.xlsx                                      ─→ IBGE socioeconomics
  └─ data/base/idhm.xlsx                                      ─→ HDI by municipality
           │
           ▼ src/ingest.py (main)
           │
  data/raw/ (per-source semicolon-delimited CSVs)
  ├─ indicadores_municipais_ma_ideb_2023.csv
  ├─ indicadores_municipais_ma_censo_escolar_2023.csv
  ├─ indicadores_municipais_ma_ibge_2010.csv
  └─ indicadores_municipais_ma_idhm_2010.csv
           │
           ▼ src/transform.py (main)
           │
  data/trusted/indicadores_municipais_ma.csv
    (48 columns: CO_MUNICIPIO, NO_MUNICIPIO, IDEB indicators,
     school infrastructure %, teacher qualification %,
     enrollment counts, IBGE socioeconomics, IDHM_2010)
           │
           ▼ src/analytics.py (main) / src/analytics.ipynb
           │
  reports/figures/*.png  +  reports/analytics-executado.html
```

## Entry Points

| Entry Point | Purpose | How to run |
|---|---|---|
| `src/ingest.py` | Ingest all sources to `data/raw/` | `uv run python src/ingest.py` |
| `src/transform.py` | Merge raw CSVs into trusted dataset | `uv run python src/transform.py` |
| `src/analytics.py` | Run full ML/stats analysis, export figures | `uv run python src/analytics.py` |
| `src/analytics.ipynb` | Interactive analysis + HTML report export | `uv run jupyter nbconvert --to html --execute src/analytics.ipynb --output analytics-executado --output-dir reports` |
| `reports/seminario_slides.qmd` | Generate RevealJS slides | `quarto render reports/seminario_slides.qmd` |
| `reports/report.qmd` | Generate Quarto HTML report | `quarto render reports/report.qmd` |

## Key Abstractions

- **Municipality as unit of analysis** — all data is aggregated to `CO_MUNICIPIO` (IBGE 7-digit code) and joined on `NO_MUNICIPIO` (name, uppercased)
- **Three-zone data lake** — `data/base/` (source/landing), `data/raw/` (per-domain cleaned), `data/trusted/` (analysis-ready merged); `data/mart/` exists but is currently empty
- **Module-level constants for model configuration** — `VARS_CLUSTER`, `VARS_EXPLICATIVAS_IDEB`, `VARS_COMPARACAO`, `CLUSTER_K` defined at top of `analytics.py` to make experiment parameters visible and easy to change
- **Cluster labeling by income rank** — clusters are named `'vulneráveis'`, `'intermediários'`, `'alta renda'` based on mean `RENDA_PER_CAPITA_2010`, making labels stable regardless of K-Means random initialization
- **Bootstrap ARI for cluster stability** — `_estabilidade_clusters()` resamples 80% of data 25 times to measure how consistent cluster assignments are
- **Render/show duality** — `_renderizar_ou_salvar(filename, salvar_arquivo)` lets every plot either save to disk (pipeline mode) or display interactively (notebook mode)
- **Proxy variables** — where direct measures are unavailable, the code constructs proxies (e.g., `PCT_MAT_15_ANOS_OU_MAIS_PROXY_DISTSERIE` as age-grade distortion proxy, `PCT_TURMAS_CORRFLUXO_FUND_AF` as dropout proxy)
