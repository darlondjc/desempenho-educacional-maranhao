# Concerns

## Technical Debt

### Incomplete `report.qmd` Template
`reports/report.qmd` is a scaffold template with multiple unfilled `TODO` markers and references to a non-existent DuckDB database file (`data/analytics.duckdb`). It imports `duckdb` and `plotly.express` (neither is in `pyproject.toml`) and references a table `fato_principal` that does not exist. Running `quarto render` on this file will crash. This file appears to be leftover boilerplate that was never tailored to the actual project.

### Dual Teacher-Ratio Computation (Inconsistency)
`src/analytics.py` (lines 63-68) recomputes `PCT_DOC_COM_SUPERIOR` and `PCT_DOC_COM_LICENCIATURA` from raw `QT_DOC_BAS` and `QT_DOC_BAS_ESCO_SUP_GRAD` columns in the trusted CSV. However, `src/ingest.py` deliberately replaces those raw counts with Sinopse Estatística 2023 data (via `build_indicadores_municipais_ma_censo_docentes_2023`) for better accuracy, producing `PCT_DOC_SUPERIOR_COMPLETO` and `PCT_DOC_LICENCIATURA_TOTAL`. The analytics script ignores the corrected columns and recalculates from the (now-replaced) raw counts, undermining the careful correction done during ingestion.

### Global Mutable State in `util.py`
`src/util.py` uses a module-level global variable `_timer_stop` (line 5) to coordinate between the timer thread and the main thread. This is not thread-safe and will produce incorrect behavior if `start_timer` is called more than once concurrently. Using a `threading.Event` would be the correct approach.

### Unused Dependencies in `pyproject.toml`
The following packages are declared in `pyproject.toml` but are not imported anywhere in `src/`:
- `duckdb` — only referenced in the incomplete `reports/report.qmd` template
- `python-dotenv` — not used anywhere
- `pandera` — declared but no schema validation is implemented
- `requests` — not used anywhere
- `seaborn` — not used anywhere

This inflates the dependency surface and the virtual environment size.

### Mixed Indentation in `src/transform.py`
`src/transform.py` uses 2-space indentation inconsistently with the rest of the project (which uses tabs in `ingest.py` and 4-space in `analytics.py`). The `main()` function at line 174 even switches to tab indentation. This style inconsistency makes the code harder to maintain.

### Notebook Contains Committed Execution Outputs
`src/analytics.ipynb` (375 KB) has all cell outputs committed to git, including embedded base64-encoded PNG images. This bloats the git history, makes diffs unreadable, and causes the notebook output to go stale as soon as any upstream data or code changes.

## Known Issues

### Index-Based Merge is Fragile (`transform.py` line 135)
`merge_dataframes` in `src/transform.py` merges the IDEB and Censo dataframes using `left_index=True, right_index=True` (positional row alignment), not on a shared key. This assumes both DataFrames have identical row order and the same municipalities at each position. While the current row counts match (217 rows each) and both happen to be alphabetically sorted by `CO_MUNICIPIO`, this assumption is implicit and undocumented. Any future change to data filtering or sorting in `ingest.py` will silently misalign municipalities.

### `visualizar_pca` Hardcodes k=2,3,4 Regardless of `k_range`
In `src/analytics.py` line 129, `visualizar_pca` iterates over `[2, 3, 4]` regardless of the `k_range` parameter passed in. If `k_range` starts at a value other than 2 or the best k falls outside that hardcoded set, the function will either crash (index out of range on `silhouettes`) or display misleading cluster plots.

### `CLUSTER_K = 2` Contradicts Three-Cluster Naming Logic
The global constant `CLUSTER_K = 2` (line 22 of `src/analytics.py`) sets k=2, but `nomear_clusters` (line 172) has explicit logic for `len(ordem) == 3` producing names like `['vulneráveis', 'intermediários', 'alta renda']`. With k=2, the three-name branch is never reached and the intermediate group never exists. The constant should be set to 3 to match the intended analytical design, or the code should be updated to make k=2 the intentional choice.

### `build_indicadores_municipais_ma_censo_escolar_2023` References a Non-Existent File
In `src/ingest.py` lines 246-260, the function attempts to load `data/raw/ideb_municipios_ma_publico.csv` to add `PCT_NAO_APROVACAO_PROXY_2023`. This file does not exist in the repository and is not generated by the pipeline (no function creates it). The code silently skips this step if the file is absent, so the column never gets populated, but there is no warning or documentation explaining this.

## Data Quality

### Temporal Mismatch: 2023 vs. 2010 Socioeconomic Data
The core clustering and regression analyses combine IDEB 2023 and Censo Escolar 2023 with IBGE income data from 2010 and IDHM from 2010 — a 13-year gap. Using 2010 socioeconomic proxies to explain 2023 educational outcomes introduces significant temporal confounding, particularly for municipalities that have undergone economic change in the intervening decade. This limitation is acknowledged in the slides but is not mitigated in the pipeline.

### String-Based Municipal Name Joins for IBGE and IDHM
The merge with IBGE and IDHM data (steps 2 and 3 in `merge_dataframes`) relies on normalized municipality name strings (`NO_MUNICIPIO` uppercased). Maranhão has municipalities with accented characters, hyphens, and spelling variations between sources (e.g., IBGE vs. INEP naming conventions). A failed join will result in `NaN` values for `RENDA_PER_CAPITA_2010` or `IDHM_2010` without any warning, silently corrupting the clustering inputs for those municipalities.

### No Validation of Post-Merge Data Completeness
Neither `src/transform.py` nor `src/analytics.py` checks for `NaN` values in the clustering variables (`RENDA_PER_CAPITA_2010`, `IDHM_2010`) after the merge. KMeans will crash or produce incorrect results if these values are missing. The `pandera` library is declared as a dependency but no schema validation is implemented anywhere.

### `MEDIA_ALUNOS_POR_TURMA_MED` Dropped Without Noting the Scope Mismatch
`src/transform.py` line 53 drops `MEDIA_ALUNOS_POR_TURMA_MED` because the study focuses on Ensino Fundamental (Anos Iniciais). However, the raw censo data includes Ensino Médio enrollments (`QT_MAT_MED`) in the aggregation and these totals flow into `QT_MAT_BAS`, potentially inflating the base counts used in `PCT_MAT_15_ANOS_OU_MAIS_PROXY_DISTSERIE`.

### Proxy Variables for Distortion and Dropout Are Approximations
`PCT_MAT_15_ANOS_OU_MAIS_PROXY_DISTSERIE` and `PCT_TURMAS_CORRFLUXO_FUND_AF` are labeled as proxies in code comments. The 15+ enrollment proxy conflates distorted-age students with adult education students at the municipal level. These limitations are not surfaced in model outputs or documented for downstream readers of the article.

## Performance

### 201 MB CSV Loaded Into Memory Without Chunking
`src/ingest.py` (`load_censo_microdados`) loads `data/base/microdados_censo_escolar_2023/dados/microdados_ed_basica_2023.csv` (201 MB, full national microdata) into a pandas DataFrame in one shot before filtering to Maranhão. On machines with limited RAM this can cause slow execution or OOM errors. The slides (`reports/seminario_slides.qmd` lines 66-69) acknowledge this as a known problem and propose DuckDB as a future solution.

### No Lazy Evaluation or Streaming in Pipeline
Every transformation step creates full DataFrame copies (`.copy()` is called in multiple places). For the current dataset size this works, but the lack of lazy evaluation (noted as a future concern in slides) means that adding more data sources or years will increase peak memory usage proportionally.

### `analytics.ipynb` is 375 KB With Binary Image Blobs
The committed notebook contains base64-encoded PNG cell outputs that bloat repository size and slow clone/diff operations. The actual analysis code in the notebook is thin (it mostly delegates to `analytics.py`), so the outputs should be stripped before committing.

## Security

### No Credentials or Secrets Detected
No API keys, passwords, or sensitive credentials were found in source files. All data sources are public government datasets accessed via local files.

### Large Raw Data File Untracked but Present on Disk
`data/base/microdados_censo_escolar_2023/dados/microdados_ed_basica_2023.csv` (201 MB) is listed as untracked in git status. The `.gitignore` does not explicitly exclude `data/base/` files, meaning this file could be accidentally committed in a future `git add -A` call, potentially pushing 201 MB of data to a remote repository. A `data/base/` exclusion rule with appropriate `!` exceptions for small reference files should be added to `.gitignore`.

### Notebook Output Contains Absolute Machine Path
`src/analytics.ipynb` contains committed cell output that includes the absolute path `/media/dados/aplicacoes/git/mestrado/Artigos/desempenho-educacional-maranhao/data/trusted/indicadores_municipais_ma.csv`, leaking the local filesystem structure of the development machine.

## Fragile Areas

### `ingest.py` and `transform.py` Use Relative Paths (`Path("data/...")`)
Both `src/ingest.py` and `src/transform.py` resolve data paths relative to the current working directory (e.g., `Path("data/base")`, `Path("data/raw")`). Running these scripts from any directory other than the project root will cause `FileNotFoundError`. `src/analytics.py` correctly uses `Path(__file__).resolve().parents[1]` — the other two modules should follow the same pattern.

### `load_sinopse_docentes_efai_2023` Hardcodes Column Positions (via `df.columns = [...]`)
In `src/ingest.py` lines 81-86, the ODS sinopse file is loaded with `header=None` and then assigned a hardcoded list of 13 column names. If INEP updates the file layout (adds or reorders columns), the assignment will silently misalign data without raising an error.

### `transform_indicadores_municipais_ma_ibge_2010` Hardcodes `iloc[:-3]`
`src/transform.py` lines 74 and 108 use `df.iloc[:-3]` to strip the last three rows from IBGE and IDHM dataframes. This assumes the source Excel files always have exactly 3 trailing footer rows. A change in the exported format will silently include garbage rows in the analysis.

### `nomear_clusters` Falls Back to Generic Names for k > 3
`src/analytics.py` lines 178-179 produce names like `'grupo 1'`, `'grupo 2'`, etc. for any k other than 2 or 3. With the elbow/silhouette loop running from k=2 to k=10, any experiment with k > 3 will produce unnamed, uninterpretable cluster labels in all downstream outputs.

## Improvement Opportunities

### Replace Index-Based Merge with Key-Based Merge
The IDEB-Censo merge in `src/transform.py` line 135 should use `on='CO_MUNICIPIO'` after normalizing both sides to the same type, rather than relying on `left_index=True, right_index=True`.

### Implement Schema Validation with Pandera
The `pandera` dependency is declared but unused. Adding a simple schema for the trusted dataset (required columns, value ranges for IDEB 0–10, percentages 0–100, non-null municipality codes) would catch pipeline errors early.

### Strip Notebook Outputs Before Committing
Using `nbstripout` as a git pre-commit hook would prevent binary image blobs from entering the repository and keep diffs readable.

### Consolidate Teacher Ratio Computation
`PCT_DOC_COM_SUPERIOR` in `src/analytics.py` should read `PCT_DOC_SUPERIOR_COMPLETO` from the trusted CSV (computed from the more accurate Sinopse source) instead of recalculating from the raw QT_ aggregates that were deliberately replaced during ingestion.

### Add `data/base/` to `.gitignore`
To prevent accidental commits of 201 MB+ raw microdata files, add a `data/base/microdados_censo_escolar_2023/dados/*.csv` or `data/base/**/*.csv` exclusion to `.gitignore`.

### Remove Unused `report.qmd` or Complete It
`reports/report.qmd` should either be removed or updated to use the actual project data and technology stack. In its current state it is non-functional and references dependencies not present in the project.
